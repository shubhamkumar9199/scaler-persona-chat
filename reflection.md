# Reflection

This assignment demonstrated the GIGO principle clearly. When prompts were vague, the outputs were generic and repetitive. After adding detailed persona descriptions, constraints, and few-shot examples, the responses became more consistent and aligned with each persona.

I learned that prompt engineering is about clarity and structure. The same model produced very different outputs depending on how the system prompt was designed. Few-shot prompting helped maintain tone and style, while constraints prevented generic responses.

Persona-based prompting made the chatbot feel realistic and purposeful. Each persona responded differently to the same question, which shows how important prompt design is.

If I had more time, I would improve realism further by studying more real-world content from each person and refining tone, vocabulary, and examples.
